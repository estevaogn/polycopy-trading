# Fase 0 — Fundação: Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bootstrap PolyCopy: Python skeleton (uv + ruff + mypy + pytest), Docker infrastructure (postgres+timescale, nats, redis, prometheus), GitHub Actions CI, pre-commit + commitizen. Zero domain code, zero agents — só a fundação.

**Architecture:** src-layout Python project (`src/polycopy/`). Single `docker-compose.yml` para os 4 serviços de infra, todos bindados em `127.0.0.1` com `environment:` explícito por serviço (least-privilege). GitHub Actions pra lint/type/test. Pre-commit + commitizen instalados como ÚLTIMO passo (hooks precisam de pyproject estável + CI verde pra não gerar reformat churn em commits iniciais).

**Tech Stack:** Python 3.12.7, uv, ruff, mypy --strict, pytest, pydantic 2.9, Docker Compose v2.18+, PostgreSQL 16 + TimescaleDB 2.17.2, NATS 2.10 + JetStream, Redis 7.4, Prometheus v2.55.1.

**Source spec:** `PROMPT_POLYCOPY_v2.md` (Fase 0, passos 0.1 a 0.5). Todas as decisões deste plano traçam pra esse documento.

**Execution model:** Usuário pede uma Task por vez (ex: "execute Task 1"). Implementador segue os steps, valida, commita. NÃO avança pra Task N+1 sem confirmação explícita do usuário (regra do prompt mestre).

```bash
#!/usr/bin/env bash
# Cria .env a partir de .env.example com POSTGRES_PASSWORD aleatório.
# Aborta se .env já existe (NÃO sobrescreve).
set -euo pipefail

if [ -f .env ]; then
    echo "ERROR: .env already exists. Delete it first if you want to regenerate." >&2
    exit 1
fi

if [ ! -f .env.example ]; then
    echo "ERROR: .env.example not found. Run from repo root." >&2
    exit 1
fi

cp .env.example .env
chmod 600 .env

PASSWORD=$(openssl rand -hex 32)
sed -i "s|^POSTGRES_PASSWORD=.*|POSTGRES_PASSWORD=${PASSWORD}|" .env

echo ".env created with generated POSTGRES_PASSWORD (chmod 600)."
```

```yaml
services:
  postgres:
    image: timescale/timescaledb:2.17.2-pg16
    restart: unless-stopped
    labels:
      com.polycopy.role: infra
    environment:
      POSTGRES_USER: ${POSTGRES_USER}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
      POSTGRES_DB: ${POSTGRES_DB}
    volumes:
      - polycopy_postgres_data:/var/lib/postgresql/data
      - ./infra/postgres/init.sql:/docker-entrypoint-initdb.d/init.sql:ro
    ports:
      - "127.0.0.1:${POSTGRES_PORT:-5432}:5432"
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U $${POSTGRES_USER} -d $${POSTGRES_DB}"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 30s

  nats:
    image: nats:2.10-alpine
    restart: unless-stopped
    labels:
      com.polycopy.role: infra
    command: ["-js", "-m", "8222"]
    volumes:
      - polycopy_nats_data:/data
    ports:
      - "127.0.0.1:4222:4222"
      - "127.0.0.1:8222:8222"
    healthcheck:
      test: ["CMD", "wget", "--quiet", "--tries=1", "--spider", "http://localhost:8222/healthz"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 10s

  redis:
    image: redis:7.4-alpine
    restart: unless-stopped
    labels:
      com.polycopy.role: infra
    command:
      - redis-server
      - --maxmemory
      - 256mb
      - --maxmemory-policy
      - allkeys-lru
      - --save
      - ""
    ports:
      - "127.0.0.1:6379:6379"
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 5s

  prometheus:
    image: prom/prometheus:v2.55.1
    restart: unless-stopped
    labels:
      com.polycopy.role: infra
    volumes:
      - ./infra/prometheus/prometheus.yml:/etc/prometheus/prometheus.yml:ro
      - polycopy_prometheus_data:/prometheus
    ports:
      - "127.0.0.1:9090:9090"
    healthcheck:
      test: ["CMD", "wget", "--quiet", "--tries=1", "--spider", "http://localhost:9090/-/ready"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 10s

volumes:
  polycopy_postgres_data:
  polycopy_nats_data:
  polycopy_prometheus_data:
```

(Notas:
- `$${POSTGRES_USER}` no healthcheck do postgres: o `$$` escapa em parse-time do compose pra `$`, então o container vê `${POSTGRES_USER}` e resolve via shell em runtime — usando a env var que setamos no bloco `environment:`.
- NATS, Redis e Prometheus NÃO têm `environment:` — não precisam de credenciais nesta fase.
- Healthchecks usam `wget --spider` (busybox) em vez de `curl` — alpine images têm wget mas geralmente não têm curl.)
